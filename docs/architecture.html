Intégration Ableton Live Agent

Boucle bidirectionnelle READ → THINK → ACT → VERIFY. Le Max for Live device collecte le contexte DAW en temps réel et exécute les actions générées par l'agent.

sequenceDiagram participant AL as 🎛️ Ableton Live 12 participant M4L as 📱 Max for Live participant WS as 🔌 WebSocket :8001 participant AG as 🧠 ReAct Agent participant LLM as 🤖 LLM Note over AL,LLM: Phase READ — Collecte du contexte DAW AL->>M4L: Live API context (tracks, devices, clips, BPM...) M4L->>WS: JSON context snapshot WS->>AG: session_context Note over AL,LLM: Phase THINK — Planification AG->>LLM: Prompt + context + tools disponibles LLM-->>AG: ReAct — Thought + Action Plan Note over AL,LLM: Phase CONFIRM — Si risque élevé AG->>M4L: action_plan + risk_level HIGH M4L->>AL: Affiche confirmation UI AL-->>M4L: Approuvé / Rejeté M4L-->>AG: user_decision Note over AL,LLM: Phase ACT — Exécution dans Ableton AG->>WS: actions array WS->>M4L: Execute actions M4L->>AL: Live API calls (set_volume, add_track, write_automation...) Note over AL,LLM: Phase VERIFY — Confirmation état final AL->>M4L: Updated state M4L->>WS: Post-action context WS->>AG: verification_snapshot AG->>LLM: Actions completed summary LLM-->>AG: Success + next suggestions
